### üéñÔ∏è ¬øQu√© es MLFQ?

**Multi-Level Feedback Queue (MLFQ)** es un algoritmo de planificaci√≥n que utiliza m√∫ltiples colas con diferentes niveles de prioridad. Los procesos se mueven din√°micamente entre estas colas seg√∫n su comportamiento:

- **Degradaci√≥n autom√°tica** cuando un proceso consume su quantum completo
- **Prevenci√≥n de inanici√≥n** mediante el envejecimiento de procesos

## üìö Algoritmos Implementados

### üîÑ Round Robin (RR)

**Descripci√≥n**: Los procesos se ejecutan en turnos rotativos durante un tiempo fijo llamado "quantum".

**Funcionamiento**:
- Cada proceso recibe un quantum de tiempo para ejecutarse
- Si el proceso no termina en su quantum, se mueve al final de la cola
- Si consume todo su quantum, se degrada al siguiente nivel
- Ideal para sistemas interactivos

```mermaid

    participant CPU
    participant P1 as Proceso 1
    participant P2 as Proceso 2

    participant Queue as Cola RR

    CPU->>P1: Ejecuta 2 unidades
    P1-->>Queue: Retorna al final
    Queue->>CPU: Asigna P2 (quantum=2)
    CPU->>P2: Ejecuta 2 unidades
    P2-->>Queue: Retorna al final
    Queue->>CPU: Asigna P3 (quantum=2)
    CPU->>P3: Ejecuta 2 unidades
    P3-->>Queue: Retorna al final
```

### ‚ö° Shortest Job First (SJF)

**Descripci√≥n**: Siempre ejecuta el proceso con menor tiempo de ejecuci√≥n restante (no apropiativo).

**Funcionamiento**:
- Selecciona el proceso con el burst time m√°s corto
- Una vez iniciado, el proceso se ejecuta hasta completarse
- Minimiza el tiempo de espera promedio
- Puede causar inanici√≥n en procesos largos

### üéØ Shortest Time to Completion First (STCF)

**Descripci√≥n**: Versi√≥n apropiativa de SJF que puede interrumpir procesos cuando llega uno m√°s corto.

**Funcionamiento**:
- Siempre ejecuta el proceso con menor tiempo restante
- Puede apropiar el CPU si llega un proceso m√°s corto
- √ìptimo para minimizar tiempo de retorno promedio
- Requiere conocimiento del tiempo de ejecuci√≥n

### üìù Formato de Archivo de Entrada

Los archivos de entrada deben seguir el siguiente formato CSV:


- `priority`: Prioridad inicial (5=m√°xima, 1=m√≠nima)
